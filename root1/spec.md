# NOVA 图书馆管理系统 - 需求规格说明书 (Spec)

## 1. 项目概述

### 1.1 项目名称
NOVA·图书馆（Cyberpunk Themed Library Management System）

### 1.2 项目目标
构建一个功能完善的校园图书馆管理系统，支持图书借阅、归还、统计、用户管理等核心功能，采用赛博朋克主题风格设计。

## 3. 非功能需求

### 3.1 性能需求
- 页面加载时间 < 2s
- 支持并发用户数 ≥ 50
- 数据库查询响应 < 500ms

### 3.2 安全需求
- 密码加密存储 (bcrypt)
- CSRF 防护
- XSS 防护
- SQL 注入防护
- 权限控制（读者/管理员/所有者）
- 会话过期机制

### 3.3 兼容性需求
- 支持主流浏览器 (Chrome, Firefox, Safari)
- 响应式设计，适配移动端

### 3.4 可用性需求
- 99.9% 系统可用性
- 友好的错误提示
- 清晰的操作指引

---

## 4. 业务边界与异常边界

### 4.1 业务边界
- **图书借阅限制**：最大借阅数 10 本（可配置）
- **借阅期限**：30 天（可配置）
- **续借次数**：1 次（可配置）
- **续借期限**：30 天（可配置）
- **每日罚款**：0.50 元/天（可配置）
- **读者权限**：仅可借阅、归还、查看个人信息
- **管理员权限**：图书管理、借阅管理、用户管理、统计查看
- **所有者权限**：全部权限

### 4.2 异常边界
- 库存不足时借阅失败
- 图书已删除时无法借阅
- 用户已冻结时无法登录/借阅
- 未登录用户无法访问需认证页面
- 无效图书ID返回404
- 重复添加相同分类名失败
- 空搜索词返回完整列表
- 超大页码返回空列表
- 无效URL格式验证失败

## 6. 数据模型

### 6.1 用户 (User)
- id
- username (唯一)
- password (加密)
- phone
- student_id
- role (reader/admin/owner)
- is_frozen
- borrow_count
- total_fine
- created_at
- updated_at

### 6.2 图书 (Book)
- id
- title
- author
- isbn
- cover
- description
- total_stock
- current_stock
- borrow_count
- is_deleted
- category (FK to Category)
- created_at
- updated_at

### 6.3 分类 (Category)
- id
- name (唯一)
- created_at
- updated_at

### 6.4 借阅记录 (BorrowRecord)
- id
- user (FK to User)
- book (FK to Book)
- borrow_date
- due_date
- return_date
- status (borrowing/returned/overdue)
- renew_count
- fine_amount
- created_at
- updated_at

### 6.5 罚款记录 (FineRecord)
- id
- user (FK to User)
- borrow_record (FK to BorrowRecord)
- amount
- status (unpaid/paid)
- paid_at
- created_at
